Under the Hood

How It Works

Three steps. Milliseconds. Production-grade reliability.

01

Paste Your URL

Submit any HTTP/HTTPS URL via the REST API or the web form. Optional: add a custom alias and TTL. Input is validated server-side using Spring Validation annotations.

POST /api/shorten
02

Backend Generates Code

Spring Boot persists the URL to Oracle SQL via JPA. The DB-assigned sequence ID is Base62-encoded (0–9, a–z, A–Z) into a 6-character code — guaranteed unique with zero collisions.

encode(id) → "aB3xZ9"
03

Share & Track

Every visit triggers an async click event (IP, User-Agent, Referer) recorded to a separate click_events table. Redis caches hot URLs. Analytics aggregate via indexed composite queries.

GET /api/analytics/{code}
Capabilities

Everything You Need

Production-ready features, not toy-project shortcuts.

Custom Aliases

Choose your own short code — url/my-product. Validated for URL-safety, uniqueness, and reserved-path conflicts server-side.

@UniqueConstraint

Click Analytics

Track every redirect: timestamp, IP, User-Agent, Referer. Aggregated into daily trends, browser breakdown, and top referrers via Oracle-native GROUP BY queries.

Composite Index

Link Expiration

Set TTL in days — expired links return HTTP 410 Gone. Expiry checked in the service layer; expired entries evicted from Redis automatically on resolution.

TTL-aware Cache

REST API

Full JSON REST API with consistent ApiResponse<T> wrapper, global @ControllerAdvice error handling, and Bean Validation on all inputs.

@RestController

Fast Redirects

HTTP 302 redirects served from a Redis-cached lookup. Cache miss falls back to an Oracle index range scan on the code column — sub-millisecond p99.

@Cacheable + Index

Scalable Data Layer

Oracle sequences for collision-free ID generation. HikariCP connection pooling. Async click recording on a dedicated thread pool. Batch inserts via saveAll().

HikariCP + @Async
